Guide d'ajout d'un departement et de concours

Projet : Plateforme concours - Universite de Ngaoundere

Ce document explique comment ajouter un nouveau departement, ses niveaux de concours, ses pieces demandees, la route de candidature et l'entree en base.

1. Comprendre l'organisation

  • prisma/schema.prisma contient les types officiels de concours dans enum ConcoursType.
  • src/lib/stab-config.ts contient les departements, niveaux, filieres, centres et pieces.
  • src/lib/concours-links.ts relie chaque type a une page de candidature.
  • src/lib/validations.ts autorise les types et departements dans les API.
  • src/app/concours/[famille]/[niveau]/page.tsx affiche le formulaire candidat.
  • La table concours contient les concours visibles par les candidats.

2. Ajouter le type dans Prisma

Ouvrir prisma/schema.prisma, puis ajouter un type dans enum ConcoursType.

enum ConcoursType {
  IAA_MAF_M1
  STAB_L1
  STAB_L3
  STAB_MASTER
  STAB_MASTER_PRO
  BIOMED_L1
  BIOMED_L3
  BIOMED_MASTER
  BIOMED_MASTER_PRO
}

Convention conseillee : DEPARTEMENT_NIVEAU, par exemple INFO_L1, INFO_M1, IAA_MAF_M1.

3. Ajouter le departement et les niveaux

Dans src/lib/stab-config.ts, ajouter le departement dans DEPARTMENT_OPTIONS.

export const DEPARTMENT_OPTIONS = [
  { value: 'IAA-MAF', label: 'IAA-MAF' },
  { value: 'STAB', label: 'STAB' },
  { value: 'Sciences biomedicales', label: 'Sciences biomedicales' },
] as const

Ensuite, associer le departement aux niveaux disponibles dans TYPE_BY_DEPARTMENT_AND_LEVEL.

const TYPE_BY_DEPARTMENT_AND_LEVEL = {
  'IAA-MAF': {
    M1: 'IAA_MAF_M1',
  },
  STAB: {
    L1: 'STAB_L1',
    L3: 'STAB_L3',
    M1: 'STAB_MASTER',
    M2: 'STAB_MASTER_PRO',
  },
}
Si un departement ne propose que M1, le formulaire admin affichera seulement M1.

4. Ajouter filieres, centres et pieces

Toujours dans src/lib/stab-config.ts, creer les constantes du nouveau concours.

export const IAA_MAF_M1_FILIERES = [
  'Applied Artificial Intelligence',
  'Financial Mathematics',
]

export const IAA_MAF_CENTERS = ['Ngaoundere', 'Yaounde (Nkolbisson)', 'Maroua']

export const IAA_MAF_M1_DOCUMENTS = [
  { type: 'demande_manuscrite_timbree', label: 'Demande manuscrite timbree', required: true },
  { type: 'formulaire_candidature', label: 'Formulaire de candidature dument rempli', required: true },
]

Chaque piece a un type technique unique, un label visible, et required pour dire si elle est obligatoire.

5. Ajouter la configuration du concours

Dans STAB_FORM_CONFIGS, ajouter une entree pour le nouveau type.

IAA_MAF_M1: {
  type: 'IAA_MAF_M1',
  niveau: 'Master professionnel 1',
  departement: 'IAA-MAF',
  title: 'Concours IAA-MAF Master professionnel 1',
  subtitle: 'Premiere annee du Master professionnel en Intelligence artificielle appliquee et Mathematiques financieres.',
  filieres: IAA_MAF_M1_FILIERES,
  centers: IAA_MAF_CENTERS,
  documents: IAA_MAF_M1_DOCUMENTS,
},

Ajouter aussi le libelle dans STAB_TYPE_LABELS.

IAA_MAF_M1: 'IAA-MAF Master professionnel 1',

6. Ajouter les champs academiques du formulaire

Dans src/components/forms/stab/AcademicSection.tsx, ajouter les champs du nouveau type.

IAA_MAF_M1: [
  { name: 'licence', label: 'Licence ou diplome equivalent' },
  { name: 'etablissement', label: 'Etablissement' },
  { name: 'anneeObtention', label: "Annee d'obtention", type: 'number' },
  { name: 'mention', label: 'Mention' },
],

7. Ajouter le lien de candidature

Dans src/lib/concours-links.ts, relier le type a sa route.

IAA_MAF_M1: '/concours/iaa-maf/m1',

8. Creer la page de candidature

Creer un fichier comme src/app/concours/iaa-maf/m1/page.tsx.

import STABMasterForm from '@/components/forms/stab/STABMasterForm'
import STABPageShell from '@/components/forms/stab/STABPageShell'
import { getCurrentUser } from '@/lib/auth'
import { getActiveStabFormConfig } from '@/lib/stab-server'

export default async function IaaMafMaster1Page() {
  const user = await getCurrentUser()
  const config = await getActiveStabFormConfig('IAA_MAF_M1')

  return (
    <STABPageShell user={user} config={config}>
      <STABMasterForm config={config} />
    </STABPageShell>
  )
}

9. Autoriser le type dans les validations

Dans src/lib/validations.ts, ajouter le type dans les enums Zod.

type: z.enum(['IAA_MAF_M1', 'STAB_L1', ...])
concoursType: z.enum(['IAA_MAF_M1', 'STAB_L1', ...])
departement: z.enum(['IAA-MAF', 'STAB', 'Sciences biomedicales'])

10. Creer le concours dans la base

Le type et la configuration ne suffisent pas : il faut aussi une ligne dans la table concours. Le plus propre est de creer un seed comme prisma/seed-iaa-maf.ts.

await prisma.concours.upsert({
  where: { id: 'iaa-maf-m1-2026' },
  update: { type: 'IAA_MAF_M1', titre: config.title, ... },
  create: { id: 'iaa-maf-m1-2026', type: 'IAA_MAF_M1', createdBy: admin.id, ... },
})

Si le concours a un PDF officiel, placer le fichier dans public/docs puis renseigner guideUrl.

guideUrl: '/docs/Concours_IAA-MAF-2026.pdf'

11. Commandes a lancer

npx prisma generate
npx prisma db push
$env:TS_NODE_COMPILER_OPTIONS='{ "module": "CommonJS" }'
npx ts-node prisma/seed-iaa-maf.ts
npm run build
npx tsc --noEmit
Sur Windows, si prisma generate echoue avec un fichier verrouille, arreter le serveur npm run dev, relancer la commande, puis redemarrer le serveur.

12. Verification finale

  • Ouvrir /concours et verifier que le nouveau departement apparait.
  • Cliquer sur Details et verifier les pieces, frais, places et filieres.
  • Verifier que Telecharger le guide apparait si guideUrl existe.
  • Cliquer sur Postuler et verifier que le formulaire charge.
  • Dans l'admin, ouvrir /admin/concours/new et verifier que le departement est disponible.
Bon reflexe : garder la configuration dans stab-config.ts comme source principale. Cela evite de disperser les filieres, centres et pieces dans plusieurs pages.
